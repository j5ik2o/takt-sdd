- [x] 1. 開発基盤と入力境界を確立する
- [x] 1.1 `cargo init` ベースの実行・検証基盤を整備する
  - RustのCLIとして起動可能な構成を初期化する
  - 探索、条件判定、出力、テスト実行に必要な依存関係を追加する
  - 標準的なテストコマンドで実行できる状態を作る
  - _Requirements: 7.1_
- [x] 1.2 入力パラメータを型安全に受理し、無効入力を開始前に失敗させる
  - ファイル名条件、内容条件、最大深度の入力を検証し、無効値で非0終了にする
  - 条件が未指定の場合は除外条件を適用しない入力状態として扱う
  - _Requirements: 1.2, 1.3, 2.2, 2.3, 6.3_

- [x] 2. 並列探索の候補収集を実装する
- [x] 2.1 常時並列で探索候補を収集し、実行ごとの再現性を保証する
  - 複数ディレクトリを含む場合でも並列で探索する
  - 実行のたびに結果集合の内容が一致するように順序特性を固定する
  - _Requirements: 4.1, 4.2_
- [x] 2.2 探索段階で無視ルールと最大深度を適用する
  - 無視ルールが存在する場合のみ探索対象から除外する
  - 無視ルールが存在しない場合は除外を適用しない
  - 最大深度Nと深度0の境界条件を満たして探索範囲を制限する
  - _Requirements: 3.1, 3.3, 6.1, 6.2_
- [x] 2.3 権限不足を記録しつつ探索を継続する
  - 読み取り不可の対象を検出しても、読み取り可能な対象の探索を止めない
  - 継続可能な問題を集約して後段で扱える状態にする
  - _Requirements: 4.3_

- [x] 3. 検索条件の適用パイプラインを実装する
- [x] 3.1 (P) ファイル名条件の一致判定を実装する
  - 条件指定時は一致した候補のみを通過させる
  - 一致件数が0件でも正常な空結果として扱う
  - _Requirements: 1.1, 1.4_
- [x] 3.2 (P) 内容条件の一致判定を実装する
  - 条件指定時は内容一致がある候補のみを通過させる
  - 一致件数が0件でも正常な空結果として扱う
  - _Requirements: 2.1, 2.4_
- [x] 3.3 条件未指定時の通過挙動と適用順序を固定する
  - 条件が未指定のときは候補を除外しない
  - 条件適用を単一路で実行し、探索条件の同時充足を保証する
  - _Requirements: 1.2, 2.2, 8.1_

- [x] 4. 出力形式の切り替えを実装する
- [x] 4.1 (P) 通常テキスト形式の出力を実装する
  - JSON指定がない場合に通常テキスト形式で出力する
  - 空結果でも正常終了の挙動を維持する
  - _Requirements: 5.2_
- [x] 4.2 (P) JSON形式の出力を実装する
  - JSON指定時にUTF-8の配列形式で出力する
  - 0件時は空配列を出力し、1件以上では各要素にパス情報を含める
  - _Requirements: 5.1, 5.3, 5.4_
- [x] 4.3 出力決定を検索結果確定後に行う
  - 条件適用後の確定結果集合のみを入力として出力形式を選ぶ
  - 出力層に探索ロジックを持ち込まない
  - _Requirements: 8.2_

- [x] 5. 検索フロー全体と終了コード契約を統合する
- [x] 5.1 探索から出力までを単一路で統合する
  - 候補収集、条件適用、出力選択を順序通りに連結する
  - 並列探索適用時にも探索条件の同時充足を崩さない
  - _Requirements: 8.1_
- [x] 5.2 空結果の正常終了と入力異常の非0終了を統一する
  - ファイル名条件0件、内容条件0件、無視対象のみの場合を正常終了として扱う
  - 入力境界で検出した無効値は非0で終了する
  - _Requirements: 1.4, 2.4, 3.2, 1.3, 2.3, 6.3_

- [x] 6. 要件充足を検証するテスト群を実装する
- [x] 6.1 (P) ユニットテストで探索条件の中核挙動を検証する
  - ファイル名条件、内容条件、無視ルール、最大深度の各境界を検証する
  - 条件未指定時に除外しない挙動と空結果の正常終了を検証する
  - _Requirements: 7.2, 1.1, 1.2, 2.1, 2.2, 3.1, 3.3, 6.1, 6.2_
- [x] 6.2 (P) 統合テストでCLI契約を検証する
  - JSON出力契約、無効な条件入力時の非0終了、最大深度境界を検証する
  - 複数回実行時の結果再現性と継続可能な探索障害時の挙動を検証する
  - _Requirements: 7.3, 5.1, 5.3, 5.4, 1.3, 2.3, 6.3, 4.2, 4.3_
- [x] 6.3 全テスト成功を品質ゲートとして確立する
  - テスト実行でユニット・統合の全ケースが成功する状態を維持する
  - テスト対象に探索条件、並列探索、出力形式を含める
  - _Requirements: 7.4, 8.3, 7.2, 7.3_
