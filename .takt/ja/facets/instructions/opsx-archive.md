完了したOpenSpec変更をアーカイブせよ。

**やること:**

1. タスクの説明から変更名を特定する
   - 名前が提供されている場合、そのまま使用する
   - 提供されていない場合、`openspec/changes/`でアクティブな変更を確認する（`archive/`ディレクトリを除外）
   - アクティブな変更が1つだけの場合、自動的に使用する
   - 複数存在する場合、リストアップしてABORTし、ユーザーに指定を求める

2. アーティファクトの完了ステータスを確認する
   ```bash
   bash scripts/opsx-cli.sh status --change "<name>" --json
   ```
   JSONを解析して以下を理解する:
   - `schemaName`: 使用されているワークフロー
   - `artifacts`: 各アーティファクトのステータスリスト

   `done`でないアーティファクトがある場合、警告として記録するが続行する。

3. タスクの完了ステータスを確認する
   変更ディレクトリ内のタスクファイル（通常`tasks.md`）を読む。
   `- [ ]`（未完了）と`- [x]`（完了）のタスクを数える。
   未完了タスクがある場合、警告として記録するが続行する。

4. デルタスペックの同期状態を評価する
   `openspec/changes/<name>/specs/`のデルタスペックを確認する。
   存在しない場合、ステップ5に進む。

   デルタスペックが存在する場合:
   - 各デルタスペックと`openspec/specs/<capability>/spec.md`の対応するメインスペックを比較する
   - 適用される変更（追加、変更、削除）を記録する
   - サマリーレポートに同期ステータスを含める

5. アーカイブを実行する
   ```bash
   mkdir -p openspec/changes/archive
   ```

   現在の日付でターゲット名を生成: `YYYY-MM-DD-<change-name>`

   ターゲットが既に存在するか確認:
   - 存在する場合: ABORTしてエラーを返し、既存アーカイブの名前変更を提案
   - 存在しない場合: 変更ディレクトリを移動
   ```bash
   mv openspec/changes/<name> openspec/changes/archive/YYYY-MM-DD-<name>
   ```

6. アーカイブの成功を確認する
   ディレクトリが新しい場所に存在し、元の場所から消えていることを確認する。

**重要なルール:**

- アーカイブ前に必ずアーティファクトとタスクの完了ステータスを確認する
- 警告でアーカイブをブロックしない - レポートに含める
- アーカイブ時に.openspec.yamlを保持する（ディレクトリと一緒に移動）
- アーカイブディレクトリの命名にYYYY-MM-DDプレフィックスを使用する
- ターゲットアーカイブディレクトリが既に存在する場合、上書きせずABORTする
